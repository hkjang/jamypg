🗂 JASQL 데이터셋 관리 콘솔

📖 관리 방법 안내 — 처음이라면 꼭 읽어주세요 (클릭해서 접기/펴기)

1데이터셋 이해하기

이 서버는 Text2SQL 정확도를 위해 18개의 JSON 데이터셋을 참조합니다. 왼쪽 목록의 뱃지 의미:

  • 필수 물리/논리 모델. 서버 기동에 필수라 제거 불가 (교체는 가능)
  • 선택 없으면 해당 기능이 축소되거나 내장 기본값 사용
  • 편집가능 이 화면에서 교체/제거 가능
  • 시스템 feedback/audit — 서버가 자동 기록. 조회만 가능
  • 이슈 N 이 파일에서 발생한 로드 오류/경고 수 (상세는 행 클릭)

2내용 확인하기

  • 목록에서 행을 클릭 → 오른쪽에 용도·기대 스키마·사용 도구·내용 샘플이 표시됩니다
  • [현재 내용 불러오기]를 누르면 파일 전체가 편집기에 로드됩니다
  • 각 데이터셋의 "기대 스키마"가 곧 작성 규칙입니다 — 그 구조 그대로 작성하세요

3넣기 / 바꾸기 (교체)

편집기 내용이 파일의 새 전체 내용이 됩니다 (부분 병합 아님 — 기존 항목 유지하려면 불러온 뒤 수정).

  • [① JSON 검사] — 브라우저에서 문법·형식(배열/객체) 사전 확인
  • [② 적용] — 서버가 자동으로: 기존 파일 백업 → 저장 → 카탈로그 재컴파일 → 핫스왑 (재기동 불필요, 즉시 반영)
  • 컴파일 실패·신규 오류 발생 시 자동 롤백 — 원본과 이전 카탈로그가 그대로 유지됩니다
  • 오류를 알고도 적용하려면 강제 적용 체크 (예: 아직 없는 테이블을 참조하는 지표 선등록)

4빼기 (제거) / 되돌리기 (복원)

  • [데이터셋 제거] — 백업 후 파일 삭제 + 핫스왑. 필수·시스템은 거부됩니다
  • 모든 교체/제거 전 상태는 백업/복원 섹션에 시각순으로 남습니다
  • [이 백업으로 복원] — 복원 직전의 현재 파일도 다시 백업되므로 복원 자체도 되돌릴 수 있습니다

5파일을 직접 수정했다면

볼륨 마운트/SSH로 data/kcb의 파일을 직접 바꾼 경우 서버는 모릅니다. 상단 [🔄 카탈로그 리로드]를 눌러 재컴파일하세요. 실패하면 이전 카탈로그가 유지됩니다.

6보안

서버를 -admin-token <값>(또는 JASQL_ADMIN_TOKEN)으로 기동하면 변경 작업에 토큰이 필요합니다. 상단 입력란에 넣으면 자동으로 전송됩니다.
