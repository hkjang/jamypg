🔌 DB 연결 관리 · 쿼리 실행

📖 사용 안내 — DB 프로파일을 등록하고, 테스트한 뒤, 아래 콘솔에서 Read-Only 쿼리를 실행합니다
  • 실행 요건: 서버가 -tags oracle로 빌드되고 실행 환경에 Oracle Instant Client가 있어야 실제 실행이 가능합니다. 상단 칩이 드라이버 없음이면 프로파일 관리·SQL 검증만 동작합니다 (docs/oracle-connector.md 참조).
  • 접속 문자열: Easy Connect host:1521/SERVICE, VIP vip-host:1521/SERVICE?connect_timeout=5&expire_time=2, TNS (DESCRIPTION=...), 암호화 tcps://... 모두 지원.
  • 비밀번호: 평문 저장 금지 — env:변수명(권장) 또는 file:/run/secrets/파일을 사용하세요. plain:값은 개발용입니다.
  • 안전장치: SELECT/WITH만 허용, DML/DDL/PLSQL/트랜잭션/DBMS_·UTL_ 차단, 다중 statement 차단, 쿼리 타임아웃, 결과 max_rows 제한(초과 시 truncated=true), 응답 바이트 캡, 연속 3회 실패 시 30초 서킷브레이커, 전 실행 감사 로그(audit/query-*.jsonl).
  • 계정: SELECT 권한만 가진 전용 계정을 사용하세요 — 서버 차단과 별개로 DB 권한이 최종 방어선입니다.
  • 실행 순서: 프로파일 등록 → [접속 테스트] 초록 확인 → 콘솔에서 ①[검증] → ②[실행계획](실제 EXPLAIN PLAN — full scan/카티션/예상행 과다를 실행 전에 탐지, high면 조건 보강) → ③[미리보기](기본 건수) → ④필요 시 [실행]. 오래 걸리면 실행 중 목록에서 [취소].
